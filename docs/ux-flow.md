# UX Flow (MVP 4차 반영)

## 기본 흐름
1. **홈(`/`)**
   - 서비스 소개 확인
   - `궁합 시작하기` 버튼으로 입력 화면 이동
2. **입력(`/match`)**
   - 사용자 A/B 정보 입력
   - 필수값 및 형식 validation 수행
   - 유효한 경우 새 `resultId` 생성
   - deterministic mock 결과 생성 후 저장(localStorage)
   - 저장 완료 후 `/result/[id]`로 이동
3. **결과(`/result/[id]`)**
   - `resultId`로 저장된 결과를 조회
   - 상단 요약(두 사람 이름/점수/관계 유형/한 줄 요약) 확인
   - `핵심 해석 → 잘 맞는 점 → 조율 포인트 → 관계 유지 팁 → 안내` 순서로 읽기
   - 공유하기/링크 복사로 URL 전달 가능
   - `정보 다시 입력하기`로 `/match` 복귀 가능

## 결과 재방문 시나리오
- 같은 브라우저에서 `/result/[id]`를 새로고침해도 동일 결과 유지
- 같은 링크를 다시 열어도 저장된 결과 재조회 가능
- 결과가 존재하지 않으면 empty state와 함께 `다시 궁합 시작하기` 버튼 노출

## 결과 페이지 정보 구조
- **요약 카드**: 결과를 빠르게 파악하는 영역
- **공유 영역**: 공유하기/링크 복사 + 복사 피드백
- **핵심 해석**: 관계를 이해하는 핵심 포인트 3개
- **잘 맞는 점**: 현재 강점 2~3개
- **조율이 필요한 점**: 충돌/주의 포인트 2~3개
- **관계 유지 팁**: 바로 적용 가능한 실천 팁 2~3개
- **안내 문구**: 참고용 해석 및 입력값에 따른 변동 가능성 안내

## 정책/안내 접근
- 홈 및 상단 네비게이션에서 `/policy/privacy`, `/policy/notice`로 이동 가능

## 추후 확장 예정
- repository 인터페이스를 유지하면서 localStorage 구현을 Supabase 구현으로 교체
- `/room`, `/join/[roomCode]`는 공동 확인 기능을 위한 예약 라우트이며,
  현재 단계에서는 "준비중" 안내만 제공한다.
