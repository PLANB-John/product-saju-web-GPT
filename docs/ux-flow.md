# UX Flow (Step 2)

## 1) 입력 페이지 `/match`
- 모바일 기준 단일 컬럼 카드 레이아웃
- 사람 1, 사람 2 섹션을 분리해 혼동 최소화
- 각 섹션 필드
  - 이름 또는 닉네임 (필수)
  - 생년월일 (필수)
  - 성별 (필수)
  - 태어난 시간 (선택, "모르면 비워도 됩니다" 안내)

## 2) Validation
- 제출 시 필수값 검사
- 사람별/필드별 짧은 한국어 메시지 노출

## 3) 결과 이동
- 입력값을 query payload에 인코딩
- deterministic ID 생성 후 `/result/[id]` 라우팅

## 4) 결과 페이지 `/result/[id]`
- 입력값 decode 후 mock generator 실행
- 노출 정보
  - 두 사람 이름
  - 궁합 점수
  - 관계 유형
  - 핵심 해석 3개
