# Product Spec (MVP 2차 반영)

## 1) 서비스 목표
- 사주 기반 궁합 서비스의 핵심 사용자 여정(홈 → 입력 → 결과)을 검증한다.
- 1차에서 만든 구조/문서를 유지하면서, 입력 폼과 최소 검증 흐름을 연결한다.

## 2) 이번 단계 범위
- Next.js App Router 기반 라우트/레이아웃 유지
- `/match` 입력 폼 구현
  - 이름
  - 생년월일
  - 출생시간/시간 미상 옵션
- 클라이언트 validation 후 결과 페이지 이동
- `/result/[id]`에서 mock 결과와 입력자 이름 확인

## 3) 다음 단계에서 할 일
- 실제 궁합 계산 로직 연결
- 결과 해석 템플릿 고도화
- room/join 협업 플로우 기획 및 구현
- 분석 이벤트/로그 수집 구조 도입
