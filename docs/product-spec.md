# Product Spec (MVP 4차 반영)

## 1) 서비스 목표
- 사주 기반 궁합 서비스의 핵심 사용자 여정(홈 → 입력 → 결과)을 검증한다.
- 기존 입력/이동 구조와 결과 데이터 흐름을 유지하면서 UI 완성도를 높인다.

## 2) 이번 단계 범위
- 홈(`/`) / 입력(`/match`) / 결과(`/result/[id]`)의 UI polish
  - 레이아웃 최대 너비, 여백, 타이포 계층 통일
  - 카드 radius/border/shadow 및 버튼 스타일(CTA/보조) 통일
  - 모바일에서 읽기 쉬운 간격/흐름 정리
- 기존 기능 로직/타입/데이터 흐름은 유지
  - `/match` validation + 결과 이동 흐름 유지
  - `/result/[id]` mock 결과 구조 유지

## 3) 제외 범위
- 실제 사주 계산 엔진 연결
- Supabase 및 외부 데이터 연동
- 결제/공유 기능 완성
- 대규모 IA/플로우 개편

## 4) 다음 단계에서 할 일
- 실제 궁합 계산 로직 연결 및 mock 교체
- 결과 섹션별 카피 정교화(데이터 기반)
- room/join 협업 플로우 기획 및 구현
- 분석 이벤트/로그 수집 구조 도입
