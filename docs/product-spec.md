# Product Spec (MVP 3차 반영)

## 1) 서비스 목표
- 사주 기반 궁합 서비스의 핵심 사용자 여정(홈 → 입력 → 결과)을 검증한다.
- 1~2차에서 만든 입력/이동 구조를 유지하면서, 결과 페이지를 읽기 쉬운 리포트 형태로 정리한다.

## 2) 이번 단계 범위
- Next.js App Router 기반 라우트/레이아웃 유지
- `/match` 입력 폼 + validation + 결과 이동 흐름 유지
- `/result/[id]` 정보 구조 개선
  - 상단 요약(이름, 점수, 관계 유형, 한 줄 요약)
  - 핵심 해석 3개
  - 잘 맞는 점 2~3개
  - 조율이 필요한 점 2~3개
  - 관계 유지 팁 2~3개
  - 안내 문구(참고용/입력값에 따른 결과 변동)
- deterministic mock 결과 타입/생성 로직 정리
  - 향후 실제 계산 엔진이 동일한 형태로 교체 가능하도록 데이터 구조 통일

## 3) 제외 범위
- 실제 사주 계산 엔진 연결
- Supabase 및 외부 데이터 연동
- 결제/공유 기능 완성
- 과도한 디자인 고도화

## 4) 다음 단계에서 할 일
- 실제 궁합 계산 로직 연결 및 mock 교체
- 결과 섹션별 카피 정교화(데이터 기반)
- room/join 협업 플로우 기획 및 구현
- 분석 이벤트/로그 수집 구조 도입
