# 사주 궁합 웹서비스 MVP

현재 구현 단계(2단계)는 **입력 경험과 기본 결과 흐름**에 집중합니다.

## 현재 가능한 사용자 흐름
1. `/match`에서 사람 1/사람 2 정보를 입력합니다.
2. 필수값(이름, 생년월일, 성별) 누락 시 사람별 한국어 validation 메시지를 표시합니다.
3. 제출하면 deterministic 결과 ID를 생성해 `/result/[id]`로 이동합니다.
4. 결과 페이지는 전달받은 입력값 기반 mock 궁합 점수/관계 유형/핵심 해석 3개를 보여줍니다.

## 범위 제외 (다음 단계)
- Supabase 연동
- 실제 사주 계산 엔진
- 공유 기능 완성
- 고도화 디자인 시스템
